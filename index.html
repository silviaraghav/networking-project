This document will guide you through the process of setting up a website with a custom domain, and demonstrating knowledge of DNS, IP addresses, and protocols.

1. Domain Name System (DNS) and Domain Configuration

The Domain Name System (DNS) is responsible for translating human-readable domain names (e.g., www.example.com) into IP addresses (e.g., 192.168.1.1) that computers can understand.

Key Concepts:

  • DNS Records: Configuring DNS records such as A Records and CNAME Records.
  • A Record: Maps a domain name to an IP address.
  • CNAME Record: Maps a subdomain to another domain.

Steps for DNS Configuration:

  1. Purchase a Domain: Buy a domain from a domain registrar (e.g., Namecheap, GoDaddy).
  2. Configure DNS Settings: Set up A Records for GitHub Pagesâ€™ IP addresses, and CNAME for subdomains like www.
  3. Verify DNS Resolution: Use a DNS lookup tool like DNSstuff to check if the domain resolves to the correct IP address.

3. IP Address and Hosting

Every device connected to the internet has a unique identifier known as an IP address. Websites are hosted on servers, each of which has an IP address that users can connect to.

Key Concepts:

  • IP Address: A numerical label that identifies a device on a network.
  • Static IP: An IP address that remains fixed.
  • Dynamic IP: An IP address that may change over time.

Steps for IP Configuration:

  1. GitHub Pages IP: GitHub Pages provides its own IP addresses. You can configure your DNS to point to GitHub's IPs:

  2. Verify IP Address: Use tools like Ping or WhatIsMyIP to confirm that the domain resolves to the correct IP address.
  3. Testing: Access the website via its domain to ensure it is live.

4. Protocols and Website Access

Websites are accessed using protocols, primarily HTTP (HyperText Transfer Protocol) and HTTPS (HyperText Transfer Protocol Secure).

Key Concepts:

  • HTTP: The standard protocol for transferring web pages. However, it is not encrypted.
  • HTTPS: A secure version of HTTP that uses SSL/TLS encryption to protect data.

Steps for Configuring HTTPS (SSL/TLS):

  1. GitHub Pages: GitHub Pages automatically supports HTTPS. Ensure that "Enforce HTTPS" is enabled in the repository settings.
  2. SSL Certificates: Use a service like Let's Encrypt to secure your domain if you are using a different hosting provider.
  3. Verify HTTPS: Check that your website loads with HTTPS (https://www.mywebsite.com) instead of HTTP.

5. Site Accessibility and Network Troubleshooting

To ensure that your website is available to all users, you must test it across different networks and devices.

Key Concepts:

  • Ping Test: Checks if the server is reachable from different locations.
  • Traceroute: Shows the path taken by data to reach your server.

Steps for Troubleshooting Network Access:

  1. Ping Your Domain: Use the ping command to test if your domain is reachable. 
    ping www.example.com
  2. Traceroute: Use the traceroute command to see how data reaches your server. 
    traceroute www.example.com
  3. Online Accessibility Tests: Use tools like Pingdom or GTMetrix to test the accessibility and performance of your website.
